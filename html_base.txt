<meta> 태그 - 문자 세트를 비롯한 문서 정보

* 문자 세트 지정하기
<meta charset="UTF-8">

* 모바일 기기 고려하기
<meta name="viewport" content="width=device-width, initial-scale=1.0">

* 인터넷 익스플로러 브라우저 고려하기
<meta http-equiv="X-UA-Compatible" content="ie=edge">

* 검색 엔진 고려하기
<meta name="keywords" content="html5, 웹표준">			해당 문서의 키워드
<meta name="description" content="html5를 통해 웹 표준 공부하기">	해당 문서의 설명
<meta name="author" content="Kyunghee Ko">				해당 문서의 소유자 또는 제작자

<body> 태그 - 실제 브라우저에 표시될 내용

블로그와 게시판에서 사용하는 HTML 태그

*위지위그 편집기
 : HTML 태그에 대해 잘 알지 못하더라도 워드나 한글 문서를 만들 듯이 마우스를 클릭해 이미지를 삽입하고 글자 크기나 색을 조절할 수 있는 편집기.
   이렇게 만들어진 소스태그들은 웹 표준을 고려하지 않고 화면에 표시된 결과만 생각해 작성되었기에 바람직한 소스는 아님.

<blockquote> 태그 - 인용문 넣기
<pre> 태그 - 입력하는 그대로 화면에 표시하기
<strong> 태그, <b> 태그 - 굵게 표시하기 (<strong>태그는 중요도 표시, <b>는 단순 구분)
<em> 태그, <i> 태그 - 이탤릭체로 표시하기 (<em> 태그는 중요도 표시, <i>는 단순 구분)
